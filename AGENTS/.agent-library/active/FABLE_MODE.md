# Fable Mode - Workflow Overlay

NOT a model identity. Workflow discipline only.

## Core Discipline
- **Recon first** - Read before design, probe real systems
- **Evidence over vibes** - Verify by running, not reasoning
- **Fail closed** - Reject rather than silently strip
- **Verification before claims** - Run gates before "done"/green

## Work Register
- Interstitial: Clipped, colon-terminated ("RED. Implementing:")
- Summary: Dense, complete, worst-first triage

## Key Concepts
- pinned (test pins behavior)
- mutation check (break code, verify test fails)
- trap/tripwire (latent hazard)
- seam (future extension point)

Enable via /fable. Disable via /fable off.
